# Google Stitch Design Rules

## Design Consistency

This project uses Google Stitch generated designs.

Always:

- Follow existing Stitch layouts
- Match current spacing system
- Match typography hierarchy
- Match existing UI patterns
- Match border radius usage
- Match shadows and card styles
- Match button styles
- Match section spacing

## Layout Rules

- Preserve existing grid systems
- Keep section alignment consistent
- Avoid changing container widths
- Use same responsive behavior

## Component Rules

Before creating new components:

- Check if Stitch already has similar component
- Reuse existing patterns
- Extend instead of redesigning

## Visual Rules

Maintain:

- Premium modern aesthetic
- Clean dashboard UI
- Minimalistic sections
- Athletic brand feeling

Avoid:

- Overdesigning
- Heavy gradients
- Inconsistent colors
- Random animations
- Different spacing systems

## Animation Rules

- Use subtle animations only
- Maintain smooth transitions
- Avoid excessive effects

## Typography Rules

- Keep existing font hierarchy
- Maintain heading consistency
- Keep readable text spacing

## Dashboard Rules

- Use clean cards
- Use analytics widgets
- Keep table styling consistent
- Use modern sidebar layouts

## Responsive Rules

- Preserve Stitch responsive behavior
- Ensure mobile optimization
- Avoid layout shifts

## Final Goal

Every new page or section should look like it was originally generated inside the same Stitch design system.
