# Momentum Fitness Frontend Rules

## Current Project Scope

Frontend only project.

## Folder Structure

- /app → pages
- /components → reusable components
- /components/ui → shared UI
- /components/dashboard → dashboard sections
- /data → mock data
- /lib → utility functions
- /types → TypeScript types

## Naming Rules

- Components → PascalCase
- Functions → camelCase
- Constants → UPPER_CASE
- Files → kebab-case

## UI Guidelines

- Premium fitness aesthetic
- Responsive layouts
- Dark/light support
- Modern dashboard design
- Reusable cards and sections
- Smooth animations

## Styling Rules

- Tailwind CSS only
- No inline styling
- Use utility-first classes
- Maintain spacing consistency

## Component Rules

- Small reusable components
- Avoid duplicated code
- Proper TypeScript interfaces
- Reusable props patterns

## Animation Rules

- Use Framer Motion
- Keep animations subtle
- Use hover effects carefully
- Smooth transitions

## Responsive Rules

- Mobile-first
- Tablet optimized
- Desktop optimized
- Avoid layout breaking

## Accessibility

- Semantic HTML
- Proper button usage
- Keyboard-friendly UI
- Proper labels

## Code Expectations

- Clean code
- Readable structure
- Production-ready frontend
- Scalable architecture
